## 🧭 UI 모형 & 사용자 흐름
각 정기점검/이관 단계에서 필요한 입력을 빠르고 정확하게 수집하도록 모바일/PC UI와 흐름을 아래와 같이 설계합니다。

### 1) 모바일 점검 스텝 (현장 작업자)
````
┌────────────────────────────────────┐
│ [1] 기본 정보 자동 입력        [GPS,PASS] │
├────────────────────────────────────┤
│ 금형 선택: AX1 / 86511-06000 / FRT UPR       │
│ 위치(GPS): 자동 캡쳐 (수동 입력 가능)        │
│ 누적 타수: 152,238 Shot                      │
│ 세척/습합 기록간 링크 ([이전 보고서])        │
└────────────────────────────────────┘
````
1-1. 기본 정보 영역은 QR/위치/금형 마스터 API로 채우고 수정 가능. shot 90% 시 자동 알림.
1-2. `정검세부 결과`에서는 위 테이블 항목을 터치하면 `사진/문서 업로드 + 확인 체크박스` 팝업이 열린다.
1-3. 작성 완료 시 `확인(인수자)` 항목으로 자동 체크인, `공정 담당` 알림 전송.

### 2) PC 대시보드 (관리자 + 인계처)
````
┌─────────────────────────────────────────────────────┐
│ 정기·이관 점검 대시보드                             │
├─────────────────────────────────────────────────────┤
│ • 관리 현황 테이블 (세척등급, 습윤, 특이사항)       │
│ • 진행률 그래프 (정검/이관 단계를 색상으로 표시)  │
│ • 최근 첨부 사진→[미리보기], 문서 → [다운로드]    │
│ • 인수 담당자 확인란: 체크박스 + 전자서명/타임스탬프│
└─────────────────────────────────────────────────────┘
````
2-1. 사진/문서 업로드는 `AWS S3`/`온프레미스` 저장소에 1:1 매핑되고, 첨부된 항목은 이력 테이블에 `category`, `filename`, `uploaded_by`, `shot_count`와 함께 저장.
2-2. 인계처 담당자는 `사진/문서 추가` 버튼으로 근거 자료를 붙여두고, `확인` 체크로 확인 완료. 거부 시 `이슈 생성` 버튼으로 유지보수 요청 연결.
2-3. UI는 청색(완료), 노랑(주의), 빨강(불량)으로 상태를 시각화하고, `이관 체크리스트`와 연동하여 `승인` 버튼이 활성화됩니다.

### 3) 자동 흐름 요약
- 현장 → 서버: 사진/영상/문서 전송 (자동 파일명 `mold-<timestamp>-<item>.jpg`).
- 서버 → 알림: 이상 상태 감지/shot 95% reached → 승인 책임자에게 `push`/`email`.
- 문서화: 완료된 점검은 PDF로 `이관 요청서`, `이관 확인서` 템플릿에 삽입되어 전자서명 요청.

## 🔗 개발 연계 자료
문서 단계에서 정리된 내용을 실제 개발에 연결할 때 참고할 수 있도록 API·DB·문서 템플릿을 각 항목별로 정리합니다.

| 영역 | 주요 항목 | 참조 문서 | 설명 |
|------|-----------|-----------|------|
| API | `/api/checklists/daily`<br>`/api/checklists/transfer`<br>`/api/checklists/photos` | `docs/DAILY_CHECK_UI_REVISED.md`<br>`docs/PERIODIC_INSPECTION_CHECKLIST.md` | 기본 정보 자동 입력, 확인 상태 업데이트, 사진 업로드/다운로드, 인수자 확인(체크박스) 등을 담당하는 REST/GraphQL 엔드포인트. 각 요청은 `mold_id`, `checklist_category`, `shot_count`, `photo_ids` 등을 payload로 포함함.
| DB | `daily_check_items`, `transfer_checklists`, `inspection_photos`, `checklist_guides` | 이 문서의 DB 스키마 섹션 (@docs/DAILY_CHECK_ITEMS.md#235-279) | 해당 테이블에 `status`, `notes`, `photo_refs`, `confirmed_by`, `confirmed_at` 등 컬럼을 포함하여 자동 집계/이관/승인 로직 구현. 외부 API와 `molds` 마스터 테이블 조인.
| 템플릿 | `TransferRequest.pdf`, `TransferConfirmation.pdf`, `InspectionSummary.xlsx` | `docs/PERIODIC_INSPECTION_CHECKLIST.md#...` (정기 점검 흐름 및 문서화 참조) | 사진/문서 첨부(미리보기 링크), 검사 결과 표, 승인자 서명란 등을 포함한 전자 문서 템플릿. PDF/Excel 자동 생성 시 `checklist_data`(JSON)와 연동하여 채움.

💡 **추가 안내**: 위 참조 문서를 기반으로 API 스펙이나 DB 마이그레이션 스크립트를 별도 `docs/API_SPEC.md` / `docs/DB_SCHEMA.md`로 작성할 수 있으며, 그 결과물은 `docs/README.md` 또는 `docs/ROADMAP.md`와 링크해 두면 좋습니다.
