## 🧭 업무 구분 및 역할 체계

금형 개발부터 폐기까지의 전체 업무는 다음 4단 역할 구조를 기준으로 수행됩니다.

```
┌─────────────────────────────────────────────────────────────┐
│              CAMS 시스템 관리 담당 (본사)                     │
│  - 시스템 규칙 관리 / 점검표 / 알람 기준 설정                  │
│  - 전사 데이터 통합 모니터링                                  │
│  - 사용자 권한 관리                                           │
│  - 시스템 설정 및 유지보수                                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                금형개발 담당 (본사)                           │
│  - 금형 기본정보 입력 및 관리                                 │
│  - 금형정보 최초 등록 시 QR 코드 자동 생성                     │
│  - 금형 사양 정의 및 승인                                     │
│  - 제작처 선정 및 관리                                        │
│  - 설계 검토 및 승인                                          │
│  - 시운전 결과 검토                                           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                  금형제작처 (Maker)                           │
│              담당자 / 작업자 (동일 ID 사용)                    │
│  - 자동생성된 QR 코드 확인                                     │
│  - QR 코드 금형 명판에 부착                                    │
│  - 금형 제작 사양 입력                                        │
│  - 제작 진행 중 세부정보 업데이트                              │
│  - 시운전(Try-out) 결과 입력                                  │
│  - 양산 가능 상태 승인 요청                                    │
│  - 수리 대응                                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                   생산처 (Plant)                             │
│              담당자 / 작업자 (동일 ID 사용)                    │
│  - 사출조건 입력                                              │
│  - 생산수량 입력 → 자동 타수 업데이트                          │
│  - 일상점검 / 정기점검                                        │
│  - 수리 요청                                                  │
│  - 세척 / 습합 관리                                           │
│  - 금형 이관 요청                                             │
│  - 폐기 요청                                                  │
└─────────────────────────────────────────────────────────────┘
```

### 권한 자동 구분
각 단계의 모든 금형 업무는 위 4단 역할 구조를 기준으로 수행되며, **QR 스캔 시 해당 사용자의 권한에 따라 입력 가능 메뉴와 화면이 자동 구분**됩니다.

- **CAMS 시스템 관리 담당**: 시스템 설정, 사용자 관리, 전체 모니터링
- **금형개발 담당**: 금형 정보 관리, QR 코드 자동 생성, 사양 승인, 제작처 관리
- **금형제작처**: QR 코드 명판 부착, 설계, 제작, 시운전, 수리
- **생산처**: 양산, 점검, 수리 요청, 이관

## 🆕 시스템 금형정보 입력 흐름

CAMS 내 금형 정보가 생성·확장·운영·보전되는 전체 데이터 입력 체계입니다.

```
┌─────────────────────────────────────────────────────────────┐
│         [CAMS 시스템 관리 담당 + 금형개발 담당]              │
│                                                               │
│  ■ CAMS 시스템 관리 담당:                                    │
│  1) 시스템 규칙 및 점검표 설정                                │
│  2) 사용자 권한 관리                                          │
│                                                               │
│  ■ 금형개발 담당:                                             │
│  1) 금형 제작 관련 기본정보(기초정보) 최초 입력                │
│     - 금형코드 / 금형명 / 부품명 / 차종                        │
│     - 제작처 / 생산처 / 목표타수                               │
│     - 금형 종류 / 캐비티 수 / 개발 단계                        │
│                                                               │
│  2) 금형정보 등록 시 QR 코드 자동 생성                         │
│     - 시스템이 자동으로 고유 QR 생성                           │
│     - QR 코드 데이터베이스 저장                                │
│                                                               │
│  3) 금형 사양 정의 및 승인                                     │
│                                                               │
│  4) 금형 제작처에 금형 정보 + QR 코드 자동 배포                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│        [금형 제작처 (담당자/작업자 공통 ID)]                   │
│                                                               │
│  1) 자동 생성된 QR 코드 확인                                   │
│     - 본사에서 배포된 QR 코드 수신                             │
│     - QR 코드 출력 (내구성 라벨)                               │
│                                                               │
│  2) 금형 제작 사양 입력                                        │
│     - 금형 2D/3D 모델 정보                                     │
│     - 냉각/슬라이드/리프터 구조                                │
│     - 금형 강재 / 핫러너 사양                                  │
│                                                               │
│  3) 금형 제작 완료 후 QR 코드 명판 부착                        │
│     - 금형 상판 (주 부착 위치)                                 │
│     - 금형 측면 (보조 부착 위치)                               │
│     - 방수/방유/내열 라벨 사용                                 │
│                                                               │
│  4) 제작 진행 중 세부정보 업데이트                             │
│                                                               │
│  5) 시운전(Try-out) 결과 입력 (PASS/FAIL)                     │
│                                                               │
│  6) 양산 가능 상태 승인 요청                                   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│          [생산처 (담당자/작업자 공통 ID)]                      │
│                                                               │
│  1) 사출조건 입력 (injection conditions)                      │
│     - 온도 / 압력 / 속도 / 냉각조건 등                         │
│                                                               │
│  2) 생산수량 입력 → 자동 타수 업데이트                         │
│                                                               │
│  3) 일상점검 (Daily Check)                                    │
│                                                               │
│  4) 정기점검 (1차/2차/3차)                                     │
│                                                               │
│  5) 수리 요청                                                  │
│                                                               │
│  6) 세척 / 습합 관리                                           │
│                                                               │
│  7) 금형 이관 요청                                             │
│                                                               │
│  8) 폐기 요청 (수명종료)                                       │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 금형 Life Cycle 전체 업무 플로어

금형 개발 → 제작 → 시운전 → 양산 → 보전/수리 → 이관 → 폐기까지의 End-to-End 업무 전체 흐름입니다.

### 전체 흐름 다이어그램

```
개발(본사) → 설계(제작처) → 제작(제작처) → 시운전 → 이관 → 양산(생산처)
                                                ↓              ↓
                                              PASS          일상점검
                                                ↓          정기점검
                                              생산            ↓
                                                ↓          수리/보전
                                              이관            ↓
                                                ↓          정상화
                                              폐기
```

### 상세 단계별 흐름
