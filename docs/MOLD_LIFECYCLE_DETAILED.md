# 금형 Lifecycle 상세 업무 흐름도

## 금형 개발부터 폐기까지 10단계 상세 프로세스

### 1단계: 금형 개발 (금형개발 담당 + CAMS 시스템 관리 담당)

```
┌──────────────────────────────────────────────────────────────┐
│  1. 금형 개발 (금형개발 담당 + CAMS 시스템 관리 담당)  │
│                                                                │
│  ■ 금형개발 담당:                                              │
│  • 부품 기획 / 금형 사양 정의                                   │
│  • 금형 제작처 선정                                             │
│  • 금형코드 생성 및 기본정보 입력                               │
│  • 금형정보 최초 등록 시 QR 코드 자동 생성                       │
│                                                                │
│  ■ CAMS 시스템 관리 담당:                                      │
│  • 시스템 규칙 및 점검표 설정                                  │
│                                                                │
│  [데이터베이스 테이블]                                          │
│  - hq_mold_specs (본사 금형 사양)                               │
│  - molds (금형 기본정보)                                        │
└──────────────────────────────────────────────────────────────┘──┘
                         │
                         ▼
```

```
└────────────────────────┬───────────────────────────────────────┘
                         │
                         ▼
```

### 3단계: 금형 제작 (제작처)

```
┌──────────────────────────────────────────────────────────────┐

│  - mold_photos (제작 사진)                                      │
└────────────────────────┬───────────────────────────────────────┘
```

```
                         │
                         ▼
```

### 6단계: 양산 생산 (생산처)

```
┌──────────────────────────────────────────────────────────────┐
│                  6. 양산 생산 (생산처)                          │
│                                                                │
│  • 일상점검 (QR + GPS + 생산수량 입력)                          │
│  • 정기점검 (1차: 100K, 2차: 500K, 3차: 1M)                     │
│  • 습합 / 세척                                                  │
│  • 타수 자동 업데이트                                           │
│  • 점검 스케줄 자동 계산                                        │
│  • 알람 발생 (임박/NG/위치이탈/타수 90%)                        │
│                                                                │
│  [데이터베이스 테이블]                                          │
│  - daily_checks (일상점검)                                      │
│  - inspections (정기점검)                                       │
│  - inspection_schedules (점검 스케줄)                           │
│  - production_quantities (생산수량)                             │
│  - shot_records (타수 기록)                                     │
│  - qr_scan_alerts (알람)                                        │
└────────────────────────┬───────────────────────────────────────┘
                         │
                         ▼
```

### 7단계: 금형 수리 (생산처/제작처)

```
┌──────────────────────────────────────────────────────────────┐
│              7. 금형 수리 (생산처/제작처)                       │
│                                                                │
│  [1] 수리 요청 발생                                             │
│  • 생산처에서 NG 발생 → 수리 요청                               │
│  • 수리 내용 및 원인 기록                                       │
│  • 수리 전 사진 첨부                                            │
│                                                                │
│  [2] 귀책 협의 프로세스                                         │
│  • 1차 협의: 생산처 ↔ 금형제작처                                │
│    - 수리 원인 분석                                             │
│    - 귀책 당사자 1차 판단                                       │
│    - 협의 내용 시스템 기록                                      │
│                                                                │
│  • 2차 협의: 본사 개입 (합의 불가 시)                           │
│    - 금형개발 담당 검토                                         │
│    - 최종 귀책 판정                                             │
│    - 비용 처리 방안 결정                                        │
│                                                                │
│  • 귀책 유형:                                                   │
│    - 생산처 귀책: 부적절한 사용, 관리 소홀                      │
│    - 제작처 귀책: 설계/제작 결함, 재질 불량                     │
│    - 자연 마모: 정상 사용 범위 내 마모                          │
│    - 기타: 불가항력, 외부 요인                                  │
│                                                                │
│  [3] 수리 진행                                                  │
│  • 귀책 합의 완료 후 수리 시작                                  │
│  • 단계별 사진 (수리 전/중/후)                                  │
│  • 수리 진행 상황 업데이트                                      │
│                                                                │
│  [4] 수리 완료 및 비용 처리                                     │
│  • 수리 완료 보고                                               │
│  • 귀책에 따른 비용 청구                                        │
│  • 수리 이력 관리                                               │
│                                                                │
│  [데이터베이스 테이블]                                          │
│  - repairs (수리 요청)                                          │
│  - repair_liability (귀책 협의)                                 │
│  - repair_photos (수리 사진)                                    │
│  - repair_costs (수리 비용)                                     │
└────────────────────────┬───────────────────────────────────────┘
                         │ 반복 (불량/알람 발생 시)
                         ▼
```

## 📊 전체 흐름 요약

```
개발 → 설계 → 제작 → 시운전 → 이관 → 양산 → 수리 → 보전 → 이관 → 폐기
 ↓      ↓      ↓       ↓       ↓      ↓      ↓      ↓      ↓      ↓
금개   제작처  제작처  제작처   양쪽   생산처  양쪽   생산처  양쪽   금개
담당                                                                    담당

* 금개담당 = 금형개발 담당 (본사)
* CAMS 시스템 관리 담당은 전 과정 모니터링
```
